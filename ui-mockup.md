## Interaction Flows

### Adding Files Flow
```
1. User Action: Drag files or click browse
2. Validation: Check file types and sizes
3. If valid: Add to queue with "Pending" status
4. If invalid: Show error toast with details
5. Update queue counter
6. Enable "Start Processing" button
```

### Processing Flow
```
1. Click "Start Processing"
2. First item status â†’ "Processing"
3. Show progress bar and time elapsed
4. On complete: Move to completed section
5. Start next item automatically
6. When all done: Show notification
```

### Error Recovery Flow
```
1. Conversion fails â†’ Show error in queue item
2. User clicks "Retry"
3. Item returns to "Pending" state
4. Reprocessed when queue reaches it
```
